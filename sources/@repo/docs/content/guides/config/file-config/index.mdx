Owing to the flexibility afforded by file based configuration is
most likely the best pick for most **bud.js** projects.

## The basics

- A **bud.js** config file must include `bud.config` in the filename.
- The file exports a single function which receives an instance of **bud**.
- The function can be async.
- If using `require`, the file should be named with the `.cjs` extension
- If using `import`, the file should be named with the `.mjs` extension
- If the project has opted-in to ESM with `{"type": "module"}`, the file can be named with the `.js` extension.

Here is a super minimal example, to give you an idea:

## Using multiple config files

You are not limited to a single configuration file.
